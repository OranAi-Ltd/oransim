# Writing a Data Provider

> **Status:** Coming soon (v0.2 with canonical schema v1.1).

## What is a Data Provider?

A **DataProvider** is a pluggable class that normalizes raw vendor data into canonical schemas for a given platform. Same platform, multiple data sources → write one provider per source.

## Interface

```python
from abc import ABC, abstractmethod
from oransim.data.schema import CanonicalKOL, CanonicalNote, CanonicalFanProfile

class DataProvider(ABC):
    @abstractmethod
    def fetch_kol(self, kol_id: str) -> CanonicalKOL: ...

    @abstractmethod
    def search_notes(self, keyword: str, **kwargs) -> list[CanonicalNote]: ...

    @abstractmethod
    def fetch_fan_profile(self, kol_id: str) -> CanonicalFanProfile: ...
```

## Built-in Providers (shipping in v0.2)

- `SyntheticProvider` — synthetic data generated by `scripts/gen_synthetic_data.py`
- `CSVProvider` — load from user-exported CSV
- `JSONProvider` — load from local JSON dumps
- `OpenAPIProvider` — any REST API conforming to OpenAPI 3.0 + mapping config

## Example: Writing a CSV Provider

```python
import pandas as pd
from oransim.platforms.xhs.providers.base import DataProvider
from oransim.data.schema import CanonicalKOL

class MyCSVProvider(DataProvider):
    def __init__(self, csv_path: str):
        self.df = pd.read_csv(csv_path)

    def fetch_kol(self, kol_id: str) -> CanonicalKOL:
        row = self.df[self.df.kol_id == kol_id].iloc[0]
        return CanonicalKOL(
            kol_id=row.kol_id,
            nickname=row.nickname,
            niche=row.niche,
            fan_count=row.fan_count,
            avg_engagement=row.avg_engagement,
            ...
        )
```
